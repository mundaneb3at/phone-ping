Claude Code → Phone Ping

Free phone push notifications whenever Claude needs your attention
~5 min setup · zero cost

The idea

Kick off a long task in Claude Code, walk away, get a buzz on your phone the moment Claude needs you back. No paid services, no API keys — one PowerShell script, three hook entries in settings.json, one free mobile app (ntfy.sh).

Flow

Triggerturn-end · permission prompt · AskUserQuestion
→
Hookping_notify.ps1
Invoke-WebRequest POST
→
Deliveryntfy.sh/
<your-topic>

The three hooks

Stop
matcher: (empty)
End of every Claude turn — most frequent ping.
Notification
matcher: (empty)
Permission prompts (Bash, PowerShell, file writes) and idle timeouts.
PreToolUse
matcher: AskUserQuestion
Every multiple-choice question. One ping per call regardless of sub-question count.
Why three? The Notification hook covers permission prompts and idle states, but AskUserQuestion is a tool call, not a notification event — it sneaks past Notification entirely. The PreToolUse entry catches it. Skip this and you'll miss every multiple-choice prompt.

Setup

  1. Install the ntfy mobile app

    ntfy by binwiederhier on Android or iOS. Free, no account. Open the app and Subscribe to a topic. Topic names on ntfy.sh are public namespace — use a random suffix so it's not guessable (e.g. my-claude-4829).

  2. Drop the hook script in place

    Copy ping_notify.ps1 to ~/.claude/hooks/ping_notify.ps1. Edit the URL line to your topic:

    -Uri "https://ntfy.sh/YOUR-TOPIC-HERE"

  3. Enable the kill switch flag

    The hook script no-ops unless this file exists. This is your on/off control.

    New-Item -ItemType File "$env:USERPROFILE\.claude\ping_enabled" -Force

  4. Wire the hooks in settings.json

    Merge into ~/.claude/settings.json under the "hooks" key (replace <USER> with your username):

    {
  "hooks": {
    "Stop": [{
      "matcher": "",
      "hooks": [{ "type": "command", "command": "powershell -NonInteractive -File \"C:\\Users\\<USER>\\.claude\\hooks\\ping_notify.ps1\"" }]
    }],
    "Notification": [{
      "matcher": "",
      "hooks": [{ "type": "command", "command": "powershell -NonInteractive -File \"C:\\Users\\<USER>\\.claude\\hooks\\ping_notify.ps1\"" }]
    }],
    "PreToolUse": [{
      "matcher": "AskUserQuestion",
      "hooks": [{ "type": "command", "command": "powershell -NonInteractive -File \"C:\\Users\\<USER>\\.claude\\hooks\\ping_notify.ps1\"" }]
    }]
  }
}

  5. Test

    Start a Claude Code session. Run whoami (or any command not in your allowlist). Phone buzzes. Let the turn end. Phone buzzes again. Done.

New-Item -ItemType File "$env:USERPROFILE\.claude\ping_enabled" -Force

toggle_ping.ps1 (in this folder) flips the state in one call.

Troubleshooting

No notifications at all

Confirm the ntfy app is subscribed to the exact topic name (case-sensitive). Confirm ~/.claude/ping_enabled exists. Run the script manually — it should silently succeed: powershell -File ~/.claude/hooks/ping_notify.ps1.

SSL error 35 / cert errors

Don't use curl on Windows in Bash/Git Bash context — stale system certs cause SSL handshake failures. The script uses PowerShell's Invoke-WebRequest for this reason. Don't swap it back.

Turn-end pings work but AskUserQuestion doesn't

The PreToolUse matcher must be exactly AskUserQuestion (case-sensitive). Check settings.json.

Turn-end pings work but permission prompts don't

The Notification hook is either missing or has a non-empty matcher. Matcher should be "" to catch all notification events.

Why not just desktop notifications?

Desktop notifications only work when your laptop is open and unlocked. Phone push works when you've walked away from the desk — which is the actual use case. Kick off a long task, go make coffee, get a buzz when Claude needs you back.
